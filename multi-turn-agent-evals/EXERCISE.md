# Lab 2 Exercise: Multi-Turn Agent Evaluation Analysis

**Total Points: 50**

## Problem 1: Run and Analyze a Multi-Turn Conversation (50 points)

### Task

Run the multi-turn evaluation suite, pick one scenario, and write a detailed
analysis of how the simulated conversation unfolded and how the scorers
evaluated it.

### Steps

1. Run the evaluation:
   ```bash
   uv run python eval.py 2>&1 | tee debug.log
   ```

2. Open `debug.log` and `metrics.txt`. Pick any one scenario from the results.

3. Write your findings in a file called `analysis.md` in this directory.

### What to Include in analysis.md

Your `analysis.md` must contain two sections:

#### Section 1: Overall Assessment

A short paragraph summarizing the overall evaluation results:
- How many scenarios passed vs failed?
- Which scorers had the highest and lowest average scores?
- Were there any patterns across personas (polite vs demanding vs confused)?

#### Section 2: Single Scenario Deep Dive

Pick any one scenario and trace through its conversation in `debug.log`.
Tell the story of what happened: what the simulated user said, how the
agent responded, which tools were called, and how the conversation
evolved turn by turn. Quote specific lines from the debug log.

Discuss how the persona traits influenced the simulated user's behavior,
whether the goal was completed, and what scores the scenario received
across all 5 scorers. End with your own take on whether the scores
were fair and if anything could be improved.

### Requirements

- **This must be written by you, not generated by AI.** We will check for
  AI-generated content. The value of this exercise is in YOU reading the
  debug log, understanding the multi-turn conversation flow, and forming
  your own analysis.
- The overall assessment must reference actual numbers from `metrics.txt`.
- The scenario deep dive must quote specific lines from `debug.log`.
- All 5 scorers must be covered with explanations for the scores.

### Tips

- Use `grep "\[Tool\]" debug.log` to filter tool calls
- Use `grep "Turn" debug.log` to see the conversation turn boundaries
- Use `grep "goal completed" debug.log` to find where goals were met
- Read `architecture.md` to understand how eval.py orchestrates the conversation loop
- The `metrics.txt` file has the same summary that prints to the console
