## Problem 1: Evaluation Analysis (40 points)

### Task

Run the evaluation suite, examine the results, and write an analysis of every case where a scorer scored below 1.0.

### Steps

1. Run the evaluation:
   ```bash
   uv run python eval.py 2>&1 | tee debug.log
   ```

2. Open `debug.log` and `eval_metrics.json`. Find every case where any scorer gave a score less than 1.0.

3. For each low-scoring case, compare the agent output against the expected output and determine why the scorer marked it down.

4. Write your findings in a file called `analysis.md` in this directory.

### What to Include in analysis.md

Your `analysis.md` must have two sections:

**1. Overall Assessment** -- A paragraph summarizing the agent's strengths and weaknesses based on the evaluation data. x 

**2. Low-Scoring Cases** -- For each case where any scorer gave a score below 1.0: 
- What was the input question?
- What did the agent output vs what did the expected output say?
- Why did the scorer give a low score?
- Is this a genuine agent failure, a dataset issue, or a scorer issue? What would you change -- the agent, the dataset, or the scorer?

### Example Entry

```markdown
### Case: "What is the capital of Australia?"
- **Scorer**: ToolSelection
- **Score**: 0.0
- **Expected tools**: ["duckduckgo_search"]
- **Actual tools used**: []
- **What happened**: The agent answered correctly (Canberra) from its
  training knowledge without calling the search tool.
- **Verdict**: Dataset issue. The expected_tools should be [] for
  questions the model can answer from training data. No agent fix needed.
```

### Requirements

- **This must be written by you, not generated by AI.** We will check for AI-generated content. The value of this exercise is in YOU reading the debug log, understanding what happened, and forming your own analysis.
- Every case where any scorer scored below 1.0 must be covered.

### Deliverables

## Problem 2: Extend the Agent with Two New Tools (60 points)

### Task

Add two new tools to the agent, write evaluation test cases for them, run the full evaluation suite, and submit the results.

### New Tools to Implement

#### Tool 1: Timezone / Current Time (30 points)

Use Python's built-in `datetime` and `zoneinfo` modules (no external API needed) to get the current time in any city.

The tool should:
- Accept a city name (e.g., "Tokyo", "New York", "London")
- Map the city name to a timezone identifier (e.g., "Tokyo" -> "Asia/Tokyo")
- Return the current local time, timezone name, and UTC offset
- Log all activity with the `[Tool]` prefix
- Handle errors gracefully (city not found)
- Be added to `tools.py` following the existing pattern

**Implementation hints**:
- Use `from zoneinfo import ZoneInfo` and `from datetime import datetime`
- Create a dictionary mapping city names to IANA timezone identifiers (e.g., `{"tokyo": "Asia/Tokyo", "london": "Europe/London"}`)
- Get current time: `datetime.now(ZoneInfo("Asia/Tokyo"))`
- Format UTC offset: `now.strftime("%z")` gives `+0900`, format as `+09:00`
- Get timezone abbreviation: `now.strftime("%Z")` gives `JST`

#### Tool 2: Currency Exchange (30 points)

Use the free [Frankfurter API](https://www.frankfurter.app/) (no API key needed) to get currency exchange rates.

**API endpoint**: `https://api.frankfurter.app/latest`

The tool should:
- Accept a base currency and target currency (e.g., USD to EUR)
- Return the current exchange rate and converted amount for a given value
- Log all activity with the `[Tool]` prefix
- Handle errors gracefully (invalid currency, API down)
- Be added to `tools.py` following the existing pattern

**Implementation hints**:
- Get latest rates: `GET https://api.frankfurter.app/latest?from=USD&to=EUR`
- The API supports standard ISO 4217 currency codes (USD, EUR, GBP, JPY, etc.)

### Steps

1. **Implement the tools** in `tools.py`:
   - Follow the existing tool pattern (private helpers at top, public @tool functions below)
   - All log messages must use the `[Tool]` prefix
   - Run `uv run python -m py_compile tools.py` to verify

2. **Register the tools** in `agent.py`:
   - Import the new tools from `tools.py`
   - Add them to the agent's `tools=[]` list
   - Update the system prompt in `prompts/system_prompt.txt` to describe the new tools

3. **Create test cases** in a new file called `dataset2.json`:
   - Create 5 new test cases that use the new tools
   - At least 2 cases should use the timezone tool
   - At least 2 cases should use the currency exchange tool
   - Include this exact test case that requires ALL 5 tools:

   ```json
   {
       "input": "I am planning a business trip from New York to Tokyo. How long is the flight route by driving to JFK first, what is the current time in Tokyo, what is the weather there, what is the USD to JPY exchange rate, and what are the top business hotels in Shinjuku?",
       "expected_output": "Driving directions from Manhattan to JFK Airport with distance and time, the current local time in Tokyo with timezone info, current Tokyo weather with temperature, the USD to JPY exchange rate, and a list of recommended business hotels in the Shinjuku area of Tokyo.",
       "expected_tools": ["get_directions", "get_current_time", "get_weather", "get_exchange_rate", "duckduckgo_search"],
       "category": "multi_tool",
       "difficulty": "hard"
   }
   ```

4. **Run the evaluation on your new dataset**:
   ```bash
   uv run python eval.py --dataset dataset2.json --output eval_metrics2.json 2>&1 | tee debug2.log
   ```

   The `--dataset` flag tells eval.py which dataset file to use, and `--output` controls where the metrics JSON is written.

5. **Take a screenshot** from Braintrust showing the 5-tool test case scores:
   - Go to your Braintrust project and click **Experiments**
   - Click on the experiment that was just created
   - Find the row for the "I am planning a business trip..." test case and click on it
   - Click the **Trace** tab to see the individual scorer results
   - Take a screenshot of that Trace view and save it as `braintrust_screenshot.png` in this directory
